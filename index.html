So what is GitHub ?

GitHub is the word that we hear most frequently nowadays, but lots of people still don't understand what is GitHub. It is not very complicate or difficult to understand GitHub. Actually, GitHub is a code hosting platform for version control and collaboration. It lets you and other users work together on a project from anywhere.

So why do we use GitHub ?

GitHub is a website which is useful in managing group's project. GitHub is a server which stores all of the project's materials, so you and your teammates can work separately at anywhere or anytime. Also, you can download or upload contents on GitHub repository, so your teammates can recieve the lastest file version easier and faster. If we don't use GitHub, we need to copy all the files in to a USB and then give that USB to your teammates, so it is time consuming and it is not an effecient way. Basic commands

So now I will introduce you some basic git commands.

1/ git clone URL

This command is use for download all the files from your GitHub repository to your computer. For the URL area, it is not the google URL. In this case, you will get the URL by clicking the green Code button in your GitHub repository.

2/ git pull

The function of this command is quite the same with "git clone". But you just use "git clone" once when you start working. Whereas, you can use "git pull" many times to make sure that you are working on an up-to-date files.

3/ git add .

"git add" is use to add all the files you are working locally before push it back to GitHub repository. There are two different types of "git add" are "git add ." and "git add index.txt". So the difference between them is "git add ." means that you will add everything locally. Whereas, "git add index.txt" means that you will choose a specific file to add.

4/ git commit -m "text"

The letter "m" stand for message and you can type anything in the "text" but I recommend you should type something that is meaningful. In conclusion, this command is use to write a message so that your teammates know what you have done in the project and this is the reason why your message must be clear so that they can understand.

5/ git push

When you have finished your work, this command will help you to push your project back to GitHub repository. You should use this after "git commit".

Resolve conflicts ?

Ways to resolve conflicts with git.

So, why do we have conflicts ?

Conflicts occurs when we forget to use "git pull" and we create a file with the same name with a file in GitHub repository. Therefore, conflict will occur. So, that why we need to use command "git pull" regularly to update our files.

How to resolve conflicts ?

It is not difficult in resolving conflict. We just need to follow the instruction of Visual Studio Code. So there will be three ways to resolve conflict (look at the picture on the right). You can choose whether to keep the local file or keep the GitHub file or keep both.

Firstly, you need to use "git pull". Then, you will get the result like in the picture. Lastly, just click the way that you want to resolve.
